# Result University Bank

🔗 **[Демо на GitHub Pages](https://lipiridis-di.github.io/Result-University-Bank/)**

Учебный проект банковского приложения на чистом JavaScript (без фреймворков). Изначально был реализован как консольное приложение, затем переработан в полноценный веб-интерфейс с сохранением данных между сессиями.

## Функционал

- Добавление нового клиента (с автоматической генерацией id)
- Пополнение баланса
- Снятие средств
- Перевод средств между клиентами
- Удаление клиента (только при нулевом балансе)
- Просмотр списка всех клиентов
- Сохранение данных в localStorage — состояние не теряется при перезагрузке страницы
- Полное управление с клавиатуры (Enter переключает между полями, применяет действие на последнем)

## Технологии

- Vanilla JavaScript (ES6+): замыкания, деструктуризация, `map`/`find`/`findIndex`
- HTML5, CSS3 (Grid Layout, CSS-переменные)
- localStorage Web API
- Без сборщиков и фреймворков — чистый браузерный JS

## Архитектура

Проект разделён на три слоя с чёткой границей ответственности:

| Файл | Ответственность |
|---|---|
| `index.js` | Бизнес-логика банка (замыкание `createBank`). Не знает о существовании DOM. |
| `ui.js` | Работа с DOM: рендеринг, обработка событий, localStorage. |
| `index.html` / `style.css` | Разметка и визуальное представление. |

### Ключевые архитектурные решения

**Единый источник правды (single source of truth).** Все данные о клиентах хранятся только в приватном состоянии банка (`clients` внутри замыкания `createBank`). UI не хранит собственной копии данных — при каждом изменении интерфейс полностью перерисовывается на основе актуального состояния банка. Это исключает рассинхронизацию между "тем, что видно на экране" и "тем, что реально сохранено".

**Инкапсуляция через замыкание.** `createBank` возвращает только публичные методы; напрямую изменить список клиентов снаружи невозможно — только через методы с встроенной валидацией (`deposit`, `withdraw`, `transfer` и т.д.). Метод `getAllClients()` возвращает **копию** данных, а не ссылку на внутренний массив, — иначе инкапсуляция была бы фиктивной.

**Автогенерация id без коллизий.** Счётчик `nextId` вычисляется как `Math.max(...clients.map(c => c.id)) + 1`, а не как `clients.length + 1` — это защищает от повторного использования id после удаления клиента из середины списка.

**Полная перерисовка + делегирование там, где оправдано.** При каждом изменении состояния список клиентов перерисовывается целиком (`innerHTML = ''` + пересборка). Обработчики событий на статичных элементах (кнопки, панель управления) навешаны один раз обычным `addEventListener`; делегирование используется точечно — там, где это упрощает код (например, единый обработчик `keydown` на всю панель управления вместо четырёх одинаковых).

## Запуск проекта

Проект не требует сборки или установки зависимостей.

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/Lipiridis-Di/Result-University-Bank.git
   ```
2. Откройте `index.html` в браузере напрямую, либо запустите через любой локальный сервер (например, расширение Live Server в VS Code).
